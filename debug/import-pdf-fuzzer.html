Inkpour — PDF import fuzzer (nobody needed this, we built it anyway)

Loads pdf.js from a CDN + src/utils.js directly — no extension, no manifest. Pick a PDF export of a chat (e.g. one printed from a mobile browser), and this reconstructs paragraph breaks from the raw text positions on the page, then fuzzes a handful of "how big a gap counts as a new paragraph" thresholds and runs each candidate through the same parseImportedText() heuristics the real import feature uses. It also tries one smarter candidate: Google's AI Search/ Mode PDF exports render your own typed query in a distinct embedded font from the answer body text, so this fingerprints which font ID follows the "AI responses may contain mistakes" disclaimer most often across the document and treats every line in that font as "You" — everything else as the answer. All candidates get scored on how conversation-shaped the result looks, and the best guess wins. Purely for the lolz; the real plugin does not need this.
